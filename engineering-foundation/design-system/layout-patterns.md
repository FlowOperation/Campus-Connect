# Layout Patterns

## Feed Pattern
- Top search, ranking, and filter cluster
- Feed switcher where relevant (`Home`, `Popular`, `Community`)
- Paginated list content
- Floating action for content creation

## Community Pattern
- Community header with title, description, join state, and member count
- Rules/about entry points
- Pinned posts block
- Community-specific sort and flair filters

## Detail Pattern
- Header metadata section
- Main content body
- Voting/action bar directly beneath post content
- Threaded interactions beneath
- Bottom composer/action bar where applicable

## Inbox Pattern
- Segmented tabs for activity, replies, and messages
- Dense notification list
- Clear unread state and quick navigation into thread detail

## Messaging Pattern
- Conversation list with recency sorting
- Chat view with sticky composer
- Context actions for report/block/mute

## Moderation Pattern
- Queue list with severity badges
- Case detail split view: evidence + action panel
- Policy reference snippets adjacent to action controls

## Composer Pattern
- Community selector at top
- Content-type switcher (text, image, link, video, poll)
- Title/body/media inputs
- Rules and posting constraints surfaced inline
- Post and save-draft actions fixed near keyboard reach

## Responsive Rules
- Mobile first base layout.
- Tablet: side-by-side panes for moderation and messaging list/detail.
- Desktop/web: optional persistent secondary panels.

## Empty/Failure Patterns
- Empty states with next-step CTA.
- Failure states with retry plus support context id.
