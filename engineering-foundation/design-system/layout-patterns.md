# Layout Patterns

## Feed Pattern
- Top search and filter cluster
- Paginated list content
- Floating action for content creation

## Detail Pattern
- Header metadata section
- Main content body
- Threaded interactions beneath
- Bottom composer/action bar where applicable

## Messaging Pattern
- Conversation list with recency sorting
- Chat view with sticky composer
- Context actions for report/block/mute

## Moderation Pattern
- Queue list with severity badges
- Case detail split view: evidence + action panel
- Policy reference snippets adjacent to action controls

## Responsive Rules
- Mobile first base layout.
- Tablet: side-by-side panes for moderation and messaging list/detail.
- Desktop/web: optional persistent secondary panels.

## Empty/Failure Patterns
- Empty states with next-step CTA.
- Failure states with retry plus support context id.
